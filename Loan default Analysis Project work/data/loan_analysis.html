Loan Default Risk Analysis Using Financial and Demographic Data

This project analyses loan default risk using customer financial and demographic information. The workflow includes data loading, quality checks, missing value treatment, duplicate removal, feature engineering, exploratory visualisation, risk segmentation, and actionable recommendations for improving loan decision-making.

Python Pandas NumPy Matplotlib Seaborn EDA Risk Segmentation

📌 Project Overview

The purpose of this project is to understand the factors associated with loan repayment and default. The analysis focuses on applicant income, coapplicant income, loan amount, credit history, loan-to-income ratio, and demographic information such as gender, marital status, education, employment type, and property area.

Business Problem

Lenders need to identify customers who are more likely to default so they can manage risk, protect revenue, and make better approval decisions.

Dataset Focus

The dataset includes customer profile details, income values, loan amount, credit history, property area, loan term, and final loan status.

Project Goal

To clean the data, engineer useful risk features, visualise patterns, and identify practical insights that support loan default risk assessment.

1️⃣ Data Loading and Initial Exploration

The first stage imports the required Python libraries, loads the loan dataset from the project data folder, previews the first rows, checks data types, and reviews summary statistics.

Import Data Analysis Libraries
Pandas and NumPy were imported for data manipulation, while Matplotlib and Seaborn were imported for visualisation. These libraries form the foundation of the analysis workflow.
Load Dataset and Preview Rows
The dataset was loaded from the data folder and previewed using the first five rows. This confirmed the presence of key fields such as CustomerID, Age, Gender, Marital_Status, Education, Employment_Type, ApplicantIncome, and loan-related variables.
Check Data Types and Missing Values
The dataset initially contained 2,100 rows and 14 columns. Several fields had missing values, including Gender, Marital_Status, LoanAmount, and Credit_History.
Summary Statistics
The descriptive statistics show the spread of age, applicant income, coapplicant income, loan amount, and loan term. Applicant income and loan amount vary widely, making them useful variables for risk analysis.

2️⃣ Data Quality Checks and Cleaning

The next stage focuses on identifying missing values, duplicate rows, and inconsistent text values. Cleaning these issues helps ensure the analysis is accurate and reliable.

Count Missing Values
Missing values were identified in Gender, Marital_Status, LoanAmount, and Credit_History. These missing values required treatment before further analysis.
Duplicate Row Check
The dataset contained 100 duplicate rows. These duplicates could bias the analysis, so they were removed.
Duplicate Removal — Before and After
Duplicate Count Before
Dataset After Removing Duplicates
Summary

After removing duplicates, the dataset reduced from 2,100 records to 2,000 records.

Key Insight

Duplicate entries can inflate category counts and distort default rate patterns.

Recommendation

Duplicate checks should always be performed before building loan risk models or making approval decisions.

Handle Missing Values
Numerical missing values were filled using the median, while categorical missing values were filled using the mode. This approach preserves the dataset size while reducing the effect of extreme values.
Preview Cleaned Dataset
After cleaning, the dataset preview shows that missing categorical values have been replaced, making the data ready for feature engineering and visual analysis.
Standardise Text Columns
Text columns were cleaned by stripping spaces, converting text to lowercase, and converting selected columns to title case for readability.
Convert Loan Status
Loan_Status was converted from text values into numeric values where 1 represents repaid and 0 represents default. This makes the target variable easier to analyse and model.
Feature Engineering
New features were created, including Total_Income and Loan_Income_Ratio. These features provide stronger indicators of borrower financial capacity and repayment pressure.
Final Cleaned Data Check
The final cleaned dataset contains 2,000 rows and 16 columns, with all required fields complete and ready for visual analysis.

4️⃣ Overall Insights and Recommendations

Key Insights
  • The dataset originally contained missing values and duplicate records that required cleaning.
  • Credit history appears to be one of the strongest indicators of loan repayment behaviour.
  • Total income alone does not fully explain loan amount or default risk.
  • Loan-to-income ratio provides a stronger measure of borrower repayment pressure.
  • Risk segmentation helps separate customers into meaningful low, medium, and high risk groups.
  • The cleaned dataset is suitable for further modelling and prediction.
Recommendations
  • Use credit history as a major feature in loan approval and risk scoring.
  • Assess loan amount relative to total income, not as a standalone value.
  • Monitor customers with high loan-to-income ratios more closely.
  • Use risk segmentation to guide lending decisions and customer support strategies.
  • Build a predictive model using cleaned financial and demographic features.
  • Review high-risk borrowers manually before approving large loans.
